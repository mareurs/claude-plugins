CODE-EXPLORER: high-performance semantic code intelligence.
Provides symbol navigation (LSP), AST analysis (tree-sitter), semantic search (embeddings), and project memory.
Tools are available automatically — no setup step needed.

READ code — use symbol tools, NEVER Read/read_file:
  list_symbols(file)                    → see structure + line numbers (ALWAYS first)
  find_symbol(name, include_body=true)  → read one symbol body
  list_functions(file)                  → quick function signatures
  read_file is LAST RESORT — only with start_line + end_line, only after symbol tools fail.

FIND code — NEVER use Grep or Glob on source files:
  Know the name  → find_symbol(pattern)
  Know a concept → semantic_search("query")
  Need callers   → find_references(name_path, file)
  Need regex     → search_pattern(pattern)

EDIT code:
  Symbol-level (preferred) → replace_symbol / insert_before_symbol / insert_after_symbol
  Line-level (know lines)  → edit_lines(path, start_line, delete_count, new_text)

OTHER LOCAL REPOSITORIES:
  Quick peek (few files)   → use absolute paths — list_dir, read_file, list_functions, search_pattern all work without switching projects
  Deep dive (symbols, refs, semantic search) → activate_project("/absolute/path") first, explore, then switch back

RULES:
  1. Structure before content — list_symbols ALWAYS before reading
  2. NEVER Read/Grep/Glob source files — use code-explorer tools instead
  3. read_file is LAST RESORT — only when symbol tools cannot get what you need,
     and ALWAYS with start_line + end_line (never read whole files)
  4. Read/Grep/Glob are ONLY for .md .json .toml .yaml config files

FALLBACK: If code-explorer tools are not responding (MCP connection issue),
Read/Grep/Glob are acceptable as a temporary workaround.
